# 📱 Projeto: Mobile App Boilerplate – Best Practices

## Visão Geral

**Objetivo**: Criar um guia de boas práticas para desenvolvimento de aplicativos mobile com **React Native + Expo** e **TypeScript**, mantendo organização, consistência e qualidade.

**Público-alvo**: Desenvolvedores mobile que buscam padronizar seu fluxo de trabalho.

**Tecnologias**: React Native + Expo, TypeScript, Zustand, React Native Unistyles, WatermelonDB

---

## 🚀 Regras de Commit - OBRIGATÓRIAS

### Commits Semânticos com Emojis

**SEMPRE** use a estrutura: `emoji tipo: descrição`

#### Tipos de commit e emojis correspondentes:

* `✨ feat:` - Nova funcionalidade
* `🐛 fix:` - Correção de bug
* `📝 docs:` - Mudanças na documentação
* `🎨 style:` - Formatação, espaços em branco, etc.
* `♻️ refactor:` - Refatoração de código
* `⚡ perf:` - Melhorias de performance
* `✅ test:` - Adição ou correção de testes
* `🔧 chore:` - Tarefas de build, ferramentas auxiliares
* `💄 ui:` - Atualizações de interface/estilo
* `🚀 deploy:` - Deploy ou configurações de produção
* `🔒 security:` - Correções de segurança
* `🌐 i18n:` - Internacionalização
* `📱 responsive:` - Melhorias de responsividade
* `♿ a11y:` - Melhorias de acessibilidade
* `📡 api:` - Integrações com APIs externas
* `📍 location:` - Funcionalidades de localização

#### Exemplos:

```
✨ feat: add user authentication flow
🐛 fix: resolve crash on login screen
🎨 style: format code with prettier
♻️ refactor: extract reusable input component
```

## 📦 Exports e Componentes React Native

* **NUNCA use `export default`** → sempre `named exports`
* **NUNCA use `React.FC` ou `FunctionComponent`** → prefira `function`
* Prefira **function declarations** para componentes e funções principais

✅ Correto:

```typescript
// components/user-profile.tsx
interface UserProfileProps {
  name: string;
  onLogout: () => void;
}

export function UserProfile({ name, onLogout }: UserProfileProps) {
  return <Text>Hello, {name}</Text>;
}
```

❌ Incorreto:

```typescript
export default function UserProfile({ name }: UserProfileProps) {
  return <Text>Hello, {name}</Text>;
}
```

## 📝 Convenções de Código

### Nomenclatura

* **PascalCase** → componentes: `UserProfile`, `LoginForm`
* **camelCase** → funções/variáveis: `fetchUserData`, `isValidEmail`
* **kebab-case** → arquivos: `user-profile.tsx`, `login-form.tsx`
* **SCREAMING_SNAKE_CASE** → constantes: `API_BASE_URL`

### Event Handlers

✅ Correto:

```typescript
const handleLogin = () => { ... }
const onUserSelect = (id: string) => { ... }
```

❌ Errado:

```typescript
const loginHandler = () => { ... }
```

## 🎨 Estilos com React Native Unistyles

* Sempre usar **Unistyles** para consistência
* Definir **temas (claro/escuro)**
* Usar **breakpoints responsivos**
* Evitar `StyleSheet.create` nativo

Exemplo:

```typescript
export const theme = {
  colors: {
    primary: '#2563EB',
    background: '#FFFFFF',
    text: '#111827',
    error: '#DC2626',
  },
  spacing: { sm: 8, md: 16, lg: 24 }
};
```

## 📱 Boas Práticas Mobile

* Usar `SafeAreaProvider`
* Implementar loading states e error boundaries
* Garantir **acessibilidade** (labels, contraste, tamanhos adequados)
* Suportar **texto grande do sistema**

---

## ⚡ Performance

* Usar `FlatList`/`SectionList` para listas grandes
* Implementar **lazy loading**
* Debounce em inputs (300ms)
* Limpar listeners em desmontagem
* Otimizar imagens e assets

## 🌐 Networking

* Timeout padrão (10s)
* Retry logic para falhas de rede
* Cache para modo offline
* Tratar erros com mensagens claras

---

## 📍 Localização

* Solicitar permissões corretamente
* Implementar fallback para GPS indisponível
* Evitar chamadas excessivas para economizar bateria

---

## ♿ Acessibilidade

* Labels para botões e inputs
* Textos com contraste adequado
* Botões com mínimo de **44x44pt**
* Suporte a screen readers
